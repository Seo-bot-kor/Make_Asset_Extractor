==================================================================
  MARS / MAKE Extractor — AR/XR 에셋 추출 프로그램 (Windows GUI)
==================================================================

VIRNECT 의 AR/XR 프로젝트 파일에서 에셋을 추출합니다.
  - .mars  (MAKE / ARES 프로젝트)  : 3D 모델·이미지·영상·오디오 + 모델 임베드 텍스처
  - .make  (XR 콘텐츠, glTF 기반)  : 임베드 텍스처/이미지 (baseColor·metallic 등 슬롯 표시)


------------------------------------------------------------------
[ 포함된 파일 ]
------------------------------------------------------------------
  mars_extractor_gui.py   ... GUI 프로그램 본체 (소스)
  build_windows.bat       ... Windows용 .exe 만드는 스크립트
  README.txt              ... 이 문서


------------------------------------------------------------------
[ Windows .exe 만들기 — 한 번만 하면 됩니다 ]
------------------------------------------------------------------

  1) 위 3개 파일을 같은 폴더에 둡니다.

  2) Python 이 없다면 먼저 설치하세요.
     - https://www.python.org/downloads/
     - 설치 화면에서 "Add Python to PATH" 체크박스를 반드시 켤 것!

  3) build_windows.bat 를 더블클릭합니다.
     - 필요한 도구(PyInstaller 등)를 자동으로 받고 빌드합니다. (1~2분)
     - 끝나면 dist 폴더가 열리고 그 안에 MarsExtractor.exe 가 있습니다.

  4) 이제 MarsExtractor.exe 하나만 있으면
     Python 이 없는 다른 Windows PC 에서도 더블클릭으로 실행됩니다.


------------------------------------------------------------------
[ 사용법 ]
------------------------------------------------------------------

  1) MarsExtractor.exe 를 실행합니다.

  2) .mars 파일을 창 안으로 끌어다 놓거나(드래그&드롭),
     [파일 선택] 버튼으로 고릅니다.

  3) [추출 시작] 을 누릅니다.

  4) 원본 .mars 파일이 있던 폴더에
     "<파일이름>_extracted" 폴더가 생기고 자동으로 열립니다.

     └ models/    : 3D 모델 (.fbx)
     └ images/    : 이미지 (.png 등)
     └ videos/    : 영상 (.mp4)
     └ audio/     : 오디오 (.wav, .mp3)
     └ textures/  : 각 모델(.fbx) 안에 박혀있던 텍스처 이미지
                    (모델 이름별 하위 폴더로 정리됨)
     └ _metadata/ : TTS 텍스트, 리소스 목록 등 JSON

  * 창의 "모델(.fbx) 안의 텍스처도 이미지로 추출" 체크박스를 끄면
    textures 폴더는 만들지 않습니다(기본은 켜짐).

  [ .make 파일의 경우 ]
     glTF 기반이라 모든 텍스처가 파일 안에 들어있어 한 번에 추출됩니다.
     models/ 폴더에 각 오브젝트(메시)가 개별 .glb 파일로 분리되어 나옵니다.
       예) 000_Plane.glb, 001_Celling.glb, 002_Wall.glb ...
       (각 파일은 그 오브젝트가 쓰는 텍스처까지 포함해 바로 열립니다.
        Blender, Windows 3D 뷰어 등에서 사용 가능)
     images/ 폴더에 텍스처가 따로 저장되며, 파일명 뒤에 용도가 표시됩니다:
        ..._baseColor.png         (베이스컬러/알베도)
        ..._metallicRoughness.png (메탈릭/러프니스)
        ..._normal.png            (노멀맵)
        ..._emissive.png          (이미시브)
        ..._occlusion.png         (앰비언트 오클루전)
     _metadata/scene.gltf.json 에 전체 씬 구조(노드/머티리얼)가 들어있습니다.

     * 참고: 원본 .make 는 모든 오브젝트가 하나의 씬으로 합쳐진
       단일 파일입니다. 위 분리는 씬 안의 메시를 오브젝트 단위로
       나눠 개별 파일로 만든 것입니다.


------------------------------------------------------------------
[ Python 으로 바로 실행하고 싶다면 (.exe 없이) ]
------------------------------------------------------------------

  명령창에서:
     python mars_extractor_gui.py

  드래그&드롭 기능까지 쓰려면:
     python -m pip install tkinterdnd2
  (설치하지 않아도 [파일 선택] 버튼으로는 잘 동작합니다.)


------------------------------------------------------------------
[ 참고 ]
------------------------------------------------------------------
  - 파일 내부의 원본 이름이 그대로 복원됩니다(한글 파일명 포함).
  - 같은 이름이 여러 개면 _1, _2 가 붙습니다.
  - 추출된 에셋은 원본 그대로의 바이너리이며 손상되지 않습니다.
